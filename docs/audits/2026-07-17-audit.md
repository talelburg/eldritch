# Repository audit — 2026-07-17

Full-repo audit: code quality, correctness, security, infrastructure, tech
debt, and tracker hygiene. This is the condensed record — **per-finding detail
(verified `file:line` evidence, failure scenarios, suggested fixes) lives in
the issues it produced (#564–#593)**; this doc preserves what the issues
don't: method, the overall verdict, what was checked and found *sound*, and
corrections to previously-documented understanding.

## Method

Nine parallel read-only domain auditors (engine dispatch ×2, engine
core/evaluator/state, card-DSL + pipeline + generated corpus, card impls,
scenarios, server + protocol, web, infra/CI/deps/docs) over all ~88k lines of
Rust plus CI, migrations, `data/`, and docs. The full CI gauntlet was run
locally (green). Every Critical/High finding — and the key Mediums — was
re-verified by hand against the code before filing; agent claims that failed
verification were corrected (see [Corrections](#corrections)). Card/scenario
fidelity was checked against `data/arkhamdb-snapshot/` (the in-repo
authority), not memory. A second pass reconciled every finding against the
issue tracker and validated all in-code `TODO(#NNN)` anchors.

## Verdict

The core is strong; problems clustered at four seams: **suspension /
elimination interleavings in the engine**, **pipeline text-parsing heuristics
that silently mis-ingest cards**, **the server's absence of
identity/versioning/limits**, and **the web transport's error
classification**. Roughly 100 findings: 1 Critical, 14 High, ~30 Medium, the
rest Low — none of the Critical/High set was previously tracked.

## Verified sound (don't re-prove these without cause)

- **Determinism:** zero `HashMap`/`HashSet` in game-core (BTree/Vec only); no
  wall-clock or I/O; all randomness through in-state `RngState` with a frozen
  chacha-stream contract test; Fisher–Yates shuffles consume indices in fixed
  order; byte-identical replay pinned by tests in three scenario files.
- **Transactional integrity:** `apply_via` clones pristine state before
  dispatch and restores state, events, and RNG position on `Rejected` — no
  handler or evaluator path can leak partial state past a rejection.
- **Enumeration↔dispatch consistency** pinned by a full OptionId round-trip
  test; `legal_actions` delegates to the same predicates the handlers use.
- **Skill-test math:** clamp-at-0, AutoFail's distinct `FailureReason`,
  commit caps, Mind-over-Matter FAQ handling — all correct.
- **Combat/encounter framework:** soak independence, simultaneous placement
  before defeat checks, AoO gating at every call site, reshuffle determinism,
  per-investigator health/clue scaling (locations), advance-flip behavior.
- **The Gathering setup:** encounter deck exact vs the snapshot (all 12
  codes/quantities), acts/agendas/doom thresholds/chaos bag all match.
- **Security posture:** fully parameterized SQL (no injection surface);
  UUIDv4 game ids; no `inner_html` anywhere (card markup parsed to typed
  segments); `ServeDir` for static files; migrations append-only with an
  embedded checksummed migrator; per-game session mutex genuinely serializes
  applies, persist-before-mutate ordering correct.
- **Content wiring:** all 47 card impls registered with no desync possible
  (`is_playable` derives from `abilities_for`); 40/47 verified text-faithful
  to the snapshot; generated corpus verified drift-free vs a pipeline rerun.
- **Workspace/infra:** per-crate `[lints] workspace = true`,
  `unsafe_code = "deny"` with a reasoned exception, MSRV == pinned toolchain,
  seven real CI jobs, exemplary `data/` provenance docs.

## Corrections

Facts the audit corrected relative to prior docs/reviews (repo docs fixed in
PR #594):

- **`apply_via` is transactional** — the old "belt-and-suspenders
  `events.clear()` only" description (and its partial-mutation caveats in
  encounter.rs/evaluator.rs/CLAUDE.md) was outdated; rejection restores
  state, events, and RNG wholesale.
- **Deduction (01039) is not text-faithful** — it spawns a second discovery
  instead of modifying the count (FAQ-confirmed; #471). An initial review
  pass wrongly rated it faithful.
- **`trapped.rs` mid-build rejection never corrupted state** — the finding's
  claimed consequence was wrong (apply_via rolls it back); only the doc and
  the validate-first convention needed fixing.
- **Soak, fast-play costs, and activated-ability AoOs all shipped** — several
  card module docs still claimed them missing.
